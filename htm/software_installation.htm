Software Installation

Installation of VideoRay CoPilot RI is a complex process. Improper installation can result in an unstable computer. If you do have an intermediate or better understanding of installing and configuring software, contact VideoRay Technical Support for assistance.

CoPilot setup currently REQUIRES that the logged in (and thus installing) user be named VideoRay. This is standard on VideoRay provisioned production machines.

This setup also requires systems using a ROV GPS Mast and DVL to have the device PAMS programmed with the standard ID's. This may NOT be the case for some devices. In which case they will HAVE to be reprogrammed with vrDeviceConf. Please contact VideoRay Technical Support for assistance if necessary.

Step-by-step Instructions

  1. Download the installation set to a temp folder on the target machine.
  2. Install VideoRay Cockpit
  3. Run the VMware-player-4.0.5-893925 to install VMware player. (This is only necessary if you do not have it installed, if you do you can press REPAIR without harming anything)
    1. Press NEXT
    2. Press NEXT
    3. Uncheck "Check for product Updates..."
    4. Press NEXT
    5. Uncheck "Help Improve VMware Player"
    6. Press NEXT
    7. Uncheck "Desktop"
    8. Uncheck "Start Menu Programs Folder"
    9. Press NEXT
    10. Press Continue
    11. Press Restart Now; wait for the reboot and login
  4. Run the VM-CoPilot-AllVersions_xx_xx_xx.exe self extracting Virtual Machine (Press RUN if asked)
    1. Wait, then press OK when prompted.
  5. Run the Setup_CoPilot, just press OK/NEXT for every dialog window
  6. Configure the VMware network
    1. Go to Control Panel from the Windows Start menu
    2. Click on Network and Sharing Center
    3. Click "Change Adapter Settings" which is on the left pane
    4. Double click on Local Area Connection Properties
    5. In the properties windows, click on Internet Protocol Version 4
    6. Click on Properties
    7. Select the "Use the following IP address" radio button
    8. Enter 192.168.1.3 in the IP address box
    9. Enter 255.255.255.0 in the Subnet mask box
    10. Press OK to dismiss the Internet protocol properties dialog
    11. Press Close to dismiss the local area network properties window
    12. Close the Network connections window
  7. Configure the VM networking
    1. Open a Windows Explorer, navigate to C:\Program Files (x86)\VMware\VMware Player
    2. Copy the Virtual Network Editor executable vmnetcfg.exe from the temp install folder (from step 1) into the VMware Player folder
    3. Run vmnetcfg.exe from the C:\Program Files (x86)\VMware\VMware Player folder
    4. Highlight VMnet8 in the white list box at the top of the window
    5. Make sure "Select the NAT (shared hosts IP address with VMs)" radio button is checked
    6. Make sure that "Connect a host virtual adapter to this network" is checked
    7. Make sure that the "Use Local DHCP service to distribute IP address to VMs" is checked
    8. Make sure that Subnet IP is set to 192.168.107.0
    9. Make sure that Subnet mask is set to 255.255.255.0
    10. Press the NAT settings button; in the NAT Settings dialog which pops up:
      1. Set "Gateway IP" to 192.168.107.2
      2. Make sure "Allow Active FTP" is checked
      3. Make sure "Allow any Organizationally Unique Identifier" is checked
      4. Set "UDP Timeout (in seconds)" to 30
      5. Press OK
    11. Press the DHCP settings button; in the DHCP settings dialog which pops up:
      1. Set "Start Address" to 192.168.107.128
      2. Set "End Address" to 192.168.107.254
      3. Set "Default lease time" to 0 days, 0 hours, 30 minutes
      4. Set "Max lease time" to 0 days, 2 hours, 0 minutes
      5. Press OK
    12. Press OK on the "Virtual Network Editor" to close the program
  8. Start Cockpit (The ROV does NOT need to be connected)
  9. Press the CoPilot button in the Cockpit Control Bar
  10. Press "Yes, I accept the terms..." in VMplayer dialog which pops up
  11. Wait for the VM to start...
  12. Select "I COPIED IT" when prompted in the VM
  13. If the warning "A USB Device that was previously..." pops up just press OK to dismiss. BE CAREFUL this may pop up behind the Virtual machine window. If the virtual machine is taking a long time to boot, try closing the VM to see if this is the case.
  14. If a software update screen pops up you can press Download and Install. You should wait for the install to complete, once finished press the Close button. The update requires that the machine be connected to the internet.
  15. Make sure the SeeByte HASP USB license key is inserted into a USB port.
  16. In the blue Virtual Machine menu bar at the top of the screen select Virtual Machine\Removable Devices\Aladdin Knowledge Hasp...\connect (disconnect from host) NOTE: you may have to do this if you move the Hasp key to a different USB port)
  17. Insure that the top monitor resolution is set to 1280x1024
    1. Right click on the Windows desktop, select Screen Resolution in the pop-up menu, if necessary change the top monitor resolution to 1280x1024
  18. Move the VM to the top monitor and set it to full screen
  19. If the the full software suite is not licensed then certain versions need to be removed:
    1. Double-Click on the Remove-Version icon.
    2. Press the number which corresponds to the version you wish to REMOVE
    3. Press Y to confirm the removal
    4. Press Y to exit the removal tool.
    5. Repeat these steps for each additional version to remove
    6. Right click on the Remove-Version icon
    7. Select "Move to Rubbish Bin" to delete the icon
  20. Copilot should be operational now. Feel free to delete the temp install folder that was created in step 1.
